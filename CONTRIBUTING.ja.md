# コントリビューションガイド

English version: [CONTRIBUTING.md](CONTRIBUTING.md)

興味を持ってくれてありがとうございます。この文書は、リポジトリ個別の
CONTRIBUTING が無い限り [@kitsuyui](https://github.com/kitsuyui) 配下の
全リポジトリに適用されます。これらのプロジェクトは一人が個人の時間で
維持しています — メンテナの時間を尊重してもらえると嬉しいです。

issue や pull request は**日本語でも英語でも**書けます。好きな方をどうぞ。

## 作る前に話す (issue-first)

- **新機能・breaking change・大きなリファクタリング**: コードを書く前に
  issue を立てて、合意を待ってください。これはあなたの時間を守るためでも
  あります。各ライブラリは意図的に小さく保っており、良いアイデアでも
  core の外で生きる方が良いものが多くあります。
- **typo や小さく明白なバグ修正**: 直接 pull request で構いません。
  先に既存 issue が無いか確認してください。
- **バグかどうか確信が持てない**: issue を立ててください。一緒に
  切り分けましょう。

### お断りするもの

無駄な労力が発生しないよう、先に書いておきます。

- 頼んでいない大規模リファクタリングや、スタイル・文法の好みの変更。
- プロジェクトのスコープ外の機能 (各プロジェクトは1つのことに集中します)。
- 上記の issue 議論を飛ばした大きな pull request。
- 最後まで見届けるつもりのない pull request。放置される PR はレビュー時間
  だけを消費します。

## メンテナに期待できること

- レビューとマージは自分のペースで行います。数日、ときには数週間かかる
  ことがあります。2週間ほど経ったら丁寧に bump してもらって構いません。
  意味のない bump はやめてください。
- マージの最終判断はメンテナが行います。断る場合は理由を説明するよう
  努めます。
- 間違って close したと思ったら、(丁寧に) 理由を教えてください。
  人間なので間違えます。

## AI の利用について

AI を活用した貢献は [AI_POLICY.ja.md](AI_POLICY.ja.md) のルールの範囲で歓迎します。
要点: 自分の貢献を自分の言葉で説明でき、人間が責任を持てること。

## 機械的な話 (意図的に短くしています)

ビルドやテストの方法はリポジトリごとに異なります。そのリポジトリの
README と CI 設定が正です。提出前に CI と同じチェックを実行してください。

- 1 PR は 1 つの変更に絞ってください。
- 挙動が変わる場合はテストを追加・更新してください。
- 貢献を提出した時点で、そのリポジトリのライセンスでの提供に同意した
  ものとみなします。

## セキュリティ

脆弱性は公開 issue に書かないでください。[SECURITY.ja.md](SECURITY.ja.md) を
見てください。
