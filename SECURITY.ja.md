## 脆弱性の報告方法

脆弱性の詳細を含む公開 issue や pull request を作成しないでください。

GitHub のプライベート脆弱性報告機能を利用してください。リポジトリページで
**Security → Report a vulnerability** から非公開で報告できます。

報告に含める情報:

- 影響を受けるリポジトリとバージョン（またはコミット）
- 再現手順、または概念実証
- 推定される影響範囲

これらは個人プロジェクトであり、スペアタイムでメンテナンスしています。
報告はできるだけ早く（通常 2 週間以内）確認し、修正と公開のタイミングをご報告者と調整します。
公開パッケージをリリースしているリポジトリについては、修正がマージされ次第、
修正リリースとセキュリティアドバイザリを公開します。
